Lookup table for every AudioTrack known to the game.

public interface IAudioRegistry

Remarks

The registry is populated at start-up by AudioCatalog and queried by IAudioService when call sites use a track id instead of a raw resource path. Treat ids as the public contract — they live as const fields on AudioCatalog and survive asset-path renames.

Implementations are expected to be case-insensitive on ids and to reject duplicate registrations explicitly so a typo in two catalog entries doesn't silently overwrite the first one.
