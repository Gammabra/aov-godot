Contract for the central audio service.

public interface IAudioService

Remarks

Defined in Core so gameplay code (skills, status effects, exploration triggers) can request audio without taking a hard reference to Godot. The adapter (AudioManager) implements this interface against AudioServer and the Godot scene tree.

All track parameters are res:// paths (or any path Godot's ResourceLoader understands). Volume parameters use the 0..1 linear scale; the adapter converts to decibels via AudioVolumeMath.

Properties

Track registry exposed for read access (UI listings, debug overlays, etc.) and for tests that need to assert what's been registered.

Play(string)

Plays the registered track with id trackId, dispatching to the right code path (music crossfade, ambient layer, or one-shot) based on the track's Bus. The per-track BaseVolumeMultiplier is applied on top of the bus volume so individual songs that were mastered too loud can be tamed at the catalog level.
