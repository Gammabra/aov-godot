High-level scene categories used to pick the right music track.

public enum MusicContext

Fields

Battle = 3

In-battle — plays BattleTheme when registered.

Exploration = 2

Exploration / world map — plays ExplorationTheme when registered.

MainMenu = 1

Main menu / title screen / settings — plays MainMenuTheme.

None = 0

No declared context. Any music currently playing fades out.

Remarks

Scenes call AudioManager.Instance?.SetMusicContext(...) on _Ready (or whenever they enter a new gameplay phase) to declare what kind of music should be playing. The manager resolves the context to a AudioCatalog id and either crossfades to that track or — if no track is registered for the context yet — fades the current music out.

This is what keeps the main-menu theme from leaking into exploration and battle: the menu scene declares MainMenu, the exploration scene declares Exploration, and the battle scene declares Battle. As long as every gameplay scene sets its own context, the audio layer always knows what should be playing — even before the exploration / battle tracks ship.
