Remarks

Pure C# data — name, portrait path, class, level, flavour text — used by HUD widgets (PlayerStatusPanel, TurnOrderQueue, VictoryScreen) to render unit identity. Does NOT carry gameplay state; that lives on IUnitSystem.

Modelled as a record class: same reference semantics as a regular class (so a unit and its HUD widget share the same instance and a mutation propagates), but with auto-generated Equals / GetHashCode based on the property values, plus a with-expression for clean profile derivation. Reference semantics are deliberate — a struct would copy on every property access and silently break unit.EntityProfile.Level = 5.

The portrait is stored as a res:// path string rather than a Godot.Texture2D: this keeps AshesofVelsingrad.Core Godot-free and unit-testable. The Godot adapter (HUD widgets, in-world sprites) calls ResourceLoader.Load<Texture2D>(profile.PortraitPath) at render time to materialise the texture.

File lives in Core/Data/ (matching the namespace) rather than Core/Systems/ — it carries no system behaviour, only data.

PortraitPath

Godot resource path to the portrait texture (e.g. res://assets/portraits/Pikachu.png). The Godot adapter loads it lazily — Core stays free of Godot.Texture2D so this class is testable without a Godot runtime. Empty string means no portrait; HUD widgets fall back to a coloured placeholder.
