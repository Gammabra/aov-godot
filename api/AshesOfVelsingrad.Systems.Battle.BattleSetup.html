Remarks

Designed to be reusable: the same battle scene can host many different encounters by swapping the EnemyUnits list. The fields are init-only properties so a setup is immutable once constructed — BattleLauncher stores the active setup and reads from it; nothing should mutate it mid-battle.

ReturnScenePath + ReturnPosition are how the "Forfeit" flow gets the player back to where they were when they triggered the fight. Treat it as a temporary checkpoint — not a save file. If you need persistent save behaviour, layer a real save system on top.
