Remarks

As with skills, items are split into a data half (ItemDefinition) and a behaviour half (IItemBehaviour resolved through ItemRegistry by BehaviourId).

Per the user's design, items are consumed from the PartyInventory (a single shared bag) and using one consumes the unit's main action — same as casting a skill.
