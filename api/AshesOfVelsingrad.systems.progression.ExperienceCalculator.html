Remarks

The feature document fixes MaxLevel = 30 and the passive-unlock schedule (3 / 15 / 22) but does not specify an XP curve. We use a smooth polynomial curve calibrated so: 

<ul><li>Level 2 requires ~50 XP (one easy fight).</li><li>Level 10 requires ~3,000 cumulative XP.</li><li>Level 30 requires ~50,000 cumulative XP (long-tail growth).</li></ul>

Designers can replace <xref href="AshesOfVelsingrad.systems.progression.ExperienceCalculator.XpRequiredForLevel(System.Int32)" data-throw-if-not-resolved="false"></xref> in one spot
without touching the rest of the progression code.
