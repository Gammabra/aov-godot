Remarks

This is the bridge between the existing inheritance-based SkillSystem contract (which UnitSystem consumes via UnitSystem.ActiveSkills) and the new data-driven architecture. New skills should never subclass SkillSystem directly — instead, create a SkillDefinition and use From(SkillDefinition) to wrap it.

The wrapper also handles common cross-cutting concerns: mana cost deduction, cooldown start, missed-cast logging, and the BattleEvents.SkillUsed event publication. Behaviours only have to worry about applying the actual effect.
