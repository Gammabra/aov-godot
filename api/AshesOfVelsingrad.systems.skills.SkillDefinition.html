Remarks

A SkillDefinition is the "what" of a skill: name, costs, range, target type, scaling, status side-effects. The "how" lives in an ISkillBehaviour referenced by BehaviourId. This split lets designers create new skills as .tres files without writing code, while letting programmers add new behaviours without touching the data.

All numerical fields are placeholders calibrated to feel reasonable. See CHANGELOG.md → Balance Notes for the full balance table.
